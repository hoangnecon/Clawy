# OpenClaw Antigravity Bridge

This repository provides a lightweight bridge to seamlessly connect **OpenClaw** with the **Antigravity Manager** proxy, overcoming context window limitations and unlocking 128,000+ token support for paid Gemini browsing sessions.

## The Problem
When using Antigravity Manager (port `8045`) as a custom API proxy for OpenClaw, the `/v1/models` endpoint does not supply a `context_window` metadata parameter. Because of this, OpenClaw's auto-discovery mechanism assigns a hardcoded default of **4096 tokens** for the model's context limit. 

This causes OpenClaw to crash on boot if the agent's prompt exceeds the minimum threshold of 16,000 tokens, throwing the error:
`Model context window too small (4096 tokens). Minimum is 16000.`

## The Solution
This bridge acts as a middleman (running on port `8046`). It intercepts the `/v1/models` response from Antigravity Manager and dynamically injects `"context_window": 128000` into every model payload. 

By tricking OpenClaw into recognizing the massive context size—and leveraging the native `openai` provider class—we can completely bypass the 4096 limit without patching OpenClaw's compiled source code. Furthermore, the installation script optimizes OpenClaw's `compaction` logic so the bot can fully take advantage of deep context without prematurely summarizing chat histories.

## Prerequisites
- **Antigravity Manager** installed and running locally on port `8045`.
- **OpenClaw CLI** installed (`openclaw`).
- **Python 3** installed.
- **jq** installed (for JSON file manipulation).

## Installation

1. Clone or download this repository.
2. Navigate to the project directory:
   ```bash
   cd openclaw-antigravity-bridge
   ```
3. Make the install script executable:
   ```bash
   chmod +x install.sh
   ```
4. Run the automated setup:
   ```bash
   ./install.sh
   ```

## What the script does:
1. Starts the Python `proxy.py` as a background process on `localhost:8046`.
2. Runs `openclaw onboard` to register the custom endpoint as the Native `openai` provider.
3. Edits `~/.openclaw/openclaw.json` to disable the aggressive `safeguard` compaction mode, replacing it with thresholds that utilize up to 90% of the 128,000 token limit.
4. Sets the primary backend to `openai/gpt-4-turbo-preview` and restarts the OpenClaw gateway.

## Troubleshooting
If you experience connection errors, check the logs generated by the proxy:
```bash
tail -f proxy.log
```

If the port is occupied, you can edit `proxy.py` and `install.sh` to choose a different `LISTEN_PORT` (e.g., `8047`).
